# GBSA 구내식당 식단표 자동화 — 1단계 진단

## 이걸 왜 만들었나

카카오톡 채널(`pf.kakao.com`)에서 매주 올라오는 식단표 이미지를 자동으로 가져오려면,
**"실제로 로그인 없이, 클라우드 서버에서 접속했을 때 이미지에 접근할 수 있는가"**를
먼저 확인해야 합니다. 이 저장소는 그 확인만을 위한 1단계입니다.

(참고: 이 코드를 작성한 Claude의 실행 환경 자체는 `pf.kakao.com` 접속이 차단되어 있어서,
직접 테스트하지 못하고 코드만 작성했습니다. 아래 절차대로 은환님이 GitHub Actions에서
직접 실행해서 결과를 확인해주셔야 합니다.)

## 실행 방법

1. 이 폴더 전체를 새 GitHub 저장소로 업로드합니다. (예: `gbsa-cafeteria-menu`)
   - GitHub 웹 UI에서 "Add file → Upload files"로 전체 업로드하시면 됩니다 (평소 하시던 방식과 동일).
2. 저장소의 **Actions** 탭으로 이동합니다.
3. 왼쪽 목록에서 `[진단] 카카오 채널 접근성 테스트` 워크플로우를 클릭합니다.
4. 오른쪽의 **Run workflow** 버튼을 눌러 수동 실행합니다.
5. 1~2분 후 실행이 끝나면, 실행 결과 페이지 하단의 **Artifacts** 섹션에서
   `diagnose-output`을 다운로드합니다.

## 결과물 확인 방법

압축을 풀면 채널별로 아래 파일들이 있습니다:

| 파일 | 확인할 것 |
|---|---|
| `gbsa.png`, `rdb_center.png`, `nano_gaeram.png` | **스크린샷을 열어서 실제로 "소식" 게시글과 식단표 이미지가 보이는지** 육안 확인 (가장 중요) |
| `*_images.txt` | 페이지 안에서 발견된 `<img>` 태그의 주소 목록. 식단표로 추정되는 이미지 URL이 있는지 확인 |
| `*_text_preview.txt` | 페이지에 실제로 어떤 텍스트가 로딩됐는지 (로그인 요구 문구, "소식" 목록 텍스트 등) |
| `summary.txt` | 3개 채널의 HTTP 상태, 이미지 개수, 로그인 요구 여부 등 요약 |

## 다음 단계는 결과에 따라 갈립니다

- ✅ **스크린샷에 식단표 이미지가 정상적으로 보임** → 2단계로 진행: 해당 이미지의 정확한 위치(셀렉터)를 특정해서 자동으로 다운로드받는 스크립트로 발전시키고, OCR(Claude Vision API)로 표를 읽어 웹페이지를 생성하는 단계로 진행합니다.
- ⚠️ **로그인 페이지로 리다이렉트되거나 빈 화면** → 로그인 세션이 필요한 구조라는 뜻입니다. 이 경우 완전 자동화는 어렵고, "담당자가 매주 이미지 1장을 업로드하면 AI가 정리해주는" 반자동 방식으로 전환하는 것이 현실적입니다.
- ⚠️ **HTTP 오류 또는 차단(403 등)** → 클라우드 IP 차단일 가능성이 높습니다. 이 경우도 반자동 방식이 대안입니다.

결과 스크린샷/텍스트를 저에게 공유해주시면, 그걸 보고 다음 단계 코드를 이어서 만들어드리겠습니다.
